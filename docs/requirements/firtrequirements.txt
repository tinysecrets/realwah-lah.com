# Wah-Lah

Production-hardened gaming hub — FastAPI + React + MongoDB, served from one container, supervised by a self-healing watchdog.

## Launch

**`make`**

That single command runs preflight, builds the image, starts MongoDB and the app at **http://localhost:8001**, and starts a persistent watchdog that auto-restarts (and, if needed, rebuilds) on health-check failures.

| Command | What it does |
|-------------------|------------------------------------------------------------------------------|
| `make` | Start everything (foreground watchdog). |
| `make start-bg` | Start everything; run the watchdog in the background. |
| `make stop` | Stop the stack and the watchdog (data preserved). |
| `make restart` | `stop` then `start`. |
| `make logs` | Tail combined container logs. |
| `make watchlog` | Tail the self-healing watchdog log. |
| `make status` | Show container + watchdog + last-known health. |
| `make clean` | Stop and purge containers, network, **and** the MongoDB volume. Destructive. |
| `make deploy` | Push a production build to Fly.io using `fly.toml`. |

## Configuration

The stack is **zero-touch on environment** — it reads from `backend/.env` if present and from the host shell otherwise. Nothing is created or overwritten. See `.env.example` for the full variable list.

For Fly.io production secrets, use `flyctl secrets set KEY=value` (see the comment block at the bottom of `fly.toml`).

## Requirements

[Docker](https://docs.docker.com/get-docker/) + `make`. Nothing else.

.github/workflows/ci.yml (new)
# =============================================================================
# Wah-Lah — Continuous Integration
# -----------------------------------------------------------------------------
# Runs on every push and pull request against main:
# 1. backend : install deps, run ruff lint, run pytest (if tests exist)
# 2. frontend : install deps, run eslint, run yarn build
# 3. docker : build the production image to prove the Dockerfile still works
# =============================================================================

name: ci

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
  workflow_dispatch:

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs:

  backend:
    name: Backend (lint + tests)
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Python 3.11
        uses: actions/setup-python@v5
        with:
          python-version: "3.11"
          cache: pip
          cache-dependency-path: backend/requirements.txt

      - name: Install backend dependencies
        working-directory: backend
        run: |
          python -m pip install --upgrade pip
          pip install -r requirements.txt

      - name: Lint with ruff (non-blocking)
        working-directory: backend
        continue-on-error: true
        run: |
          pip install ruff
          ruff check . --output-format=github

      - name: Run pytest (if tests present)
        working-directory: backend
        run: |
          if [ -d tests ] && [ -n "$(ls -A tests 2>/dev/null)" ]; then
            pytest -q --maxfail=5
          else
            echo "No backend tests — skipping."
          fi

  frontend:
    name: Frontend (lint + build)
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node 20
        uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: yarn
          cache-dependency-path: frontend/yarn.lock

      - name: Install frontend dependencies
        working-directory: frontend
        run: yarn install --frozen-lockfile --network-timeout 600000

      - name: Lint (non-blocking)
        working-directory: frontend
        continue-on-error: true
        run: |
          if [ -f .eslintrc.js ] || [ -f .eslintrc.json ] || [ -f eslint.config.js ]; then
            yarn eslint src --max-warnings=0 || true
          else
            echo "No ESLint config — skipping."
          fi

      - name: Build production bundle
        working-directory: frontend
        env:
          CI: "false"
          REACT_APP_BACKEND_URL: ""
        run: yarn build

  docker:
    name: Docker image build
    runs-on: ubuntu-latest
    needs: [backend, frontend]
    steps:
      - uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Build image (no push)
        uses: docker/build-push-action@v6
        with:
          context: .
          file: ./Dockerfile
          push: false
          cache-from: type=gha
          cache-to: type=gha,mode=max

# =============================================================================
# Wah-Lah — Production deploy to Fly.io
# -----------------------------------------------------------------------------
# Trigger: every push to main OR manual dispatch.
# Auth: repo secret FLY_API_TOKEN (generate via `flyctl auth token`).
# Config: ./fly.toml in the repo root.
# =============================================================================

name: deploy

on:
  push:
    branches: [main]
  workflow_dispatch:

concurrency:
  group: deploy-prod
  cancel-in-progress: false

jobs:
  deploy:
    name: Fly.io rolling deploy
    runs-on: ubuntu-latest
    environment:
      name: production
      url: https://wah-lah.com
    steps:
      - uses: actions/checkout@v4

      - name: Setup flyctl
        uses: superfly/flyctl-actions/setup-flyctl@master

      - name: Deploy to Fly
        env:
          FLY_API_TOKEN: ${{ secrets.FLY_API_TOKEN }}
        run: |
          flyctl deploy \
            --config fly.toml \
            --remote-only \
            --strategy rolling \
            --wait-timeout 300

      - name: Post-deploy health probe
        run: |
          for i in $(seq 1 12); do
            if curl -fsS --max-time 5 https://wah-lah.com/api/health >/dev/null; then
              echo "✅ Production healthy."
              exit 0
            fi
            echo "Attempt $i/12 — waiting 10s..."
            sleep 10
          done
          echo "❌ Production failed to report healthy after deploy."
          exit 1
